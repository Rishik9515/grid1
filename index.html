How It Works

Three Steps to Save Lives

Deploy in under 5 minutes. Every phone in range automatically joins the network and begins detecting survivor signals.

📡
Step 01
Deploy & Connect
Run the Node.js server on any laptop. Rescue team members open the dashboard URL in Chrome. Each phone instantly registers as a mesh node via Socket.IO and claims a GPS position on the live map.
🎤
Step 02
Detect Survivors
The microphone continuously analyzes audio for tapping patterns, SOS Morse code, and voice. The accelerometer detects human movement. Every signal is confidence-scored and plotted on the map instantly.
🗺
Step 03
Coordinate & Respond
The command dashboard shows all devices and detections on a live GPS map. Click any device marker to see its full detection history. Broadcast SOS alerts to every phone simultaneously with one button.

Technology

Built on Real Standards

No proprietary APIs. No vendor lock-in. Every component is open, documented, and battle-tested.

🟢
Node.js + Express
Lightweight HTTP server. Handles REST API, WebSocket upgrades, and static file serving. Runs on any machine with Node ≥ 16.
Backend
⚡
Socket.IO
Real-time bidirectional communication. Detections, device registrations, and SOS alerts are pushed to all clients instantly over WebSockets.
Real-Time
🗄
SQLite3
Zero-configuration embedded database. Stores detections, devices, and messages with indexed queries. File-based — no separate DB server needed.
Storage
🗺
Leaflet.js
Open-source interactive maps. Renders device markers, detection points, and real-time GPS position on OpenStreetMap tiles.
Frontend Map
🎤
Web Audio API
Native browser audio analysis. FFT-based frequency analysis for tap, voice, and SOS detection. No native app required.
Sensor
🔵
Web Bluetooth API
Discovers nearby BLE devices directly from Chrome. Used for peer discovery and device registration in the mesh network.
Connectivity
📍
Geolocation API
Continuous high-accuracy position tracking via watchPosition(). Updates device markers and detection coordinates in real-time.
Location
⚙️
Service Worker
Caches all assets for offline use. Background sync queues detections when offline and replays them when connectivity returns.
Offline PWA

Deploy

Go Live in 5 Minutes

Two paths to deployment. Local for testing, cloud for real operations with HTTPS (required for Bluetooth and GPS on phones).

⚡ Quickest Path
Local + ngrok
Run on your laptop. Use ngrok to expose it over HTTPS so phones can connect with full Bluetooth and GPS support.
1
Download & extract the zip
unzip rescuemesh-complete.zip
2
Install dependencies
cd rescuemesh/backend && npm install
3
Start the server
node server.js
4
Expose with ngrok (for phones)
ngrok http 3000
⬇ Download rescuemesh-complete.zip
☁️ Production
Render.com (Free)
Deploy to the cloud for a permanent HTTPS URL. Free tier on Render works perfectly for field operations.
1
Push project to GitHub
git init && git push origin main
2
Create Web Service on Render
render.com → New → Web Service
3
Set build config
Root: backend/  |  Start: node server.js
4
Open your live HTTPS URL
https://rescuemesh.onrender.com
